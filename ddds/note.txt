================================================================================
                       DDDS v2.0 - DEVELOPER NOTES
================================================================================

1. CALIBRATION METRICS
   - Duration: The calibrator gathers Eye Aspect Ratio (EAR) samples for the first 
     90 frames (~3 seconds at 30 fps).
   - Baseline: Calculated as the mean of the captured 90 frames.
   - Drowsiness Threshold: Set to 75% of the baseline EAR (baseline * 0.75).

2. FATIGUE SCORING MODEL (0 - 100)
   - Primary weighting is distributed across three sub-scores:
     * EAR Score (60%): Measures eye closure levels.
       - EAR < 70% baseline: 100
       - EAR < 80% baseline: 70
       - EAR < 90% baseline: 40
       - Otherwise: 0
     * MAR Score (25%): Measures yawning.
       - MAR > 0.65: 90
       - MAR > 0.55: 50
       - Otherwise: 0
     * Blink Score (15%): Measures blink rate per minute.
       - < 8 blinks/min (micro-sleeps): 70
       - > 30 blinks/min (fatigue struggle): 50
       - Otherwise: 0

3. MULTI-LEVEL ALERT THRESHOLDS
   - GREEN (Score < 40): Normal driving state. Thin green border (3px).
   - AMBER (Score 40 - 64): Mild fatigue. Medium orange border (6px), console log warning.
   - RED (Score >= 65): Critical drowsiness. Thick red border (10px) with red tint overlay,
     continuous audio alarms (8s cooldown), console warnings.

4. WEB APP MJPEG STREAMING (http://localhost:5000)
   - Video feed is served via a multipart/x-mixed-replace boundary stream.
   - Background detection thread updates a shared global state dict under thread locks.
   - AJAX polling updates stats metrics in the browser DOM every 500ms.
   - Camera releases resources and closes face mesh models gracefully on shutdown.

5. TROUBLESHOOTING CAMERA ACCESS
   - If the webcam returns "Camera locked or not found", check for other active
     processes (e.g. background python tasks) holding the webcam index 0.
   - Terminate duplicate processes in the terminal, then click "Start Session".
